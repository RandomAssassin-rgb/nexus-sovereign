# Nexus Sovereign Core Rules

## Hard constraints
- Do not pivot the product away from Nexus Sovereign.
- Preserve the current worker/admin architecture.
- Preserve weekly pricing.
- Preserve income-loss-only scope.
- Do not add health, life, accident, or vehicle repair coverage.
- Prefer additive changes over rewrites.
- Do not break grouped public `/api/*` contracts unless absolutely necessary.

## Product north star
Nexus Sovereign should evolve into an insurer-grade Event Twin + Signal Fabric system.

### Event Twin
A disruption becomes a live event object containing:
- trigger source and threshold crossed
- affected zone footprint
- exposed worker cohort
- projected payout load
- fraud-risk distribution
- reserve and loss-ratio impact
- explainable payout traces
- audit-ready evidence

### Signal Fabric
The event evidence layer combines:
- weather
- historical weather
- AQI
- traffic
- simulated platform activity
- device trust
- multivariate verification
- crowd consensus
- freshness/confidence scoring
- contradiction checks
